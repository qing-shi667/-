# 左侧 OCR 数据填充设计

## 目标

将图片选择和 OCR 识别控件从右侧 AI 回答助手移动到左侧牛顿环计算页。OCR 完成后由 Zeabur 后端调用 OCR 与 DeepSeek，将识别结果整理成每行一组的 `k d` 数据并自动写入实验数据文本框，但不自动执行曲率半径与不确定度计算。

同时确保 GitHub Pages 使用真实后端地址：

```text
https://xn--11x61a012f.zeabur.app/api/chat
```

## 界面

- 左侧计算页在“实验数据：环级数 k 暗环直径 d(mm)”区域内显示图片选择控件和“OCR识别”按钮。
- OCR 状态和错误显示在左侧控件附近。
- 右侧 AI 回答助手删除图片选择和 OCR 按钮，仅保留问答功能。
- 原计算表单、示例数据、计算按钮和结果显示保持不变。

## 数据流

1. 用户在左侧选择实验数据图片。
2. 用户点击“OCR识别”。
3. 主页面读取左侧 iframe 中选择的文件并上传到 Zeabur `/api/ocr`。
4. 后端使用 Tesseract 提取文字，并调用 DeepSeek 将文字规范为 `k d` 行。
5. 主页面把 `data_text` 写入左侧 `dataInput` 文本框并显示成功状态。
6. 页面不调用 `runFit()`；用户检查数据后手动点击“开始计算+不确定度”。

## 错误处理

- 未选择图片时提示用户先选择图片。
- 请求期间禁用 OCR 按钮，避免重复上传。
- OCR 或 AI 整理失败时保留原有实验数据并在左侧显示错误。
- 成功后显示已填入的数据行数，方便用户核对。

## 验证

- 自动测试确认 OCR 控件只存在于左侧计算页。
- 自动测试确认 `fillCalculatorData` 不调用 `runFit()`。
- 自动测试确认前端使用真实 Zeabur 地址，不含占位域名。
- 浏览器验证 OCR 后数据进入左侧文本框且计算结果区域仍为空，直到手动点击计算按钮。
