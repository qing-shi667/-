# IBM / Carbon Style Design Guide

## Visual Theme
- Enterprise technical, calm, precise, and data-first.
- White surfaces, thin neutral borders, structured blue accents.
- Use dense but readable layouts, closer to a control panel than a landing page.

## Color Palette
- `#ffffff` for primary surfaces.
- `#f6f8fa` for page background.
- `#d0d7de` and `#d8dee8` for separators and field borders.
- `#24292f` and `#57606a` for text hierarchy.
- `#0969da` for primary actions and selected states.
- `#fd8c73` for subtle active underline accents.

## Typography
- Prefer system UI / Microsoft YaHei / PingFang SC.
- Headlines are compact and functional, not decorative.
- Keep letter spacing at zero.

## Components
- Buttons: rectangular, 6px radius, clear primary vs secondary contrast.
- Tabs: simple text tabs with a single thin underline for the active state.
- Panels: white background, thin border, little or no shadow.
- Inputs: soft white fields with narrow borders and generous padding.

## Layout
- Use wide desktop content with one main work area and one supporting side panel.
- Avoid heavy gradients, blobs, or large decorative cards.
- Keep spacing consistent and restrained.

## Responsive Behavior
- On mobile, collapse to a single-column tool layout.
- Use sticky top navigation and clear view switching.
- Keep controls large enough for touch, but do not inflate spacing excessively.

## Do
- Make the UI feel like a reliable technical instrument.
- Favor structure, alignment, and scanability.
- Keep the current calculator workflow intact.

## Don't
- Don't use playful gradients or decorative hero sections.
- Don't stack floating cards inside more floating cards.
- Don't increase visual noise just to fill space.
